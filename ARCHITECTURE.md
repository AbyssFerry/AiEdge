## 🔧 技术栈详解

### 1. **Web 框架层**
- **FastAPI**: 现代高性能 Web 框架
  - 自动 API 文档生成 (Swagger/OpenAPI)
  - 类型校验和序列化 (Pydantic)
  - 异步请求支持 (asyncio)
  - CORS 中间件支持跨域

- **Uvicorn**: ASGI 服务器
  - 高性能异步服务器
  - 支持 HTTP/1.1 和 WebSocket

### 2. **AI 模型层**
- **llama-cpp-python**: Llama.cpp Python 绑定
  - C++ 内核，高性能推理
  - 支持 GGUF 格式量化模型
  - GPU 加速 (n_gpu_layers=-1)
  - KQV offloading 优化
  - 原生 OpenAI 兼容 API

### 3. **数据持久层**
- **SQLite**: 轻量级关系型数据库
  - 无需独立服务器
  - 支持事务和 ACID
  - 表结构:
    - `models`: 模型元数据
    - `upload_sessions`: 上传会话管理
    - `upload_chunks`: 分片上传记录

- **SQLAlchemy**: ORM 框架 (可选)
  - 数据库抽象层
  - 连接池管理

### 4. **文件管理层**
- **Hugging Face Hub**: 模型仓库集成
  - 自动下载模型文件
  - 支持断点续传
  - 进度追踪

- **文件系统操作**:
  - `pathlib.Path`: 路径管理
  - `shutil`: 文件操作
  - 分片上传机制 (10MB/chunk)

### 5. **进程管理层**
- **subprocess**: 子进程控制
  - 启动/停止子服务
  - 进程状态监控
  - 独立进程隔离

### 6. **配置管理**
- **python-dotenv**: 环境变量管理
  - .env 文件加载
  - 配置集中化管理

### 7. **系统监控**
- **psutil**: 系统资源监控
  - CPU、内存使用率
  - 磁盘空间查询
  - GPU 显存监控 (jetson-stats)

### 8. **HTTP 客户端**
- **httpx**: 异步 HTTP 客户端
  - 支持 HTTP/2
  - 连接池和超时管理

### 9. **数据验证**
- **Pydantic**: 数据模型和验证
  - 类型安全
  - 自动文档生成
  - 数据序列化/反序列化

## 🔄 核心工作流程

### 1. **启动流程**
```
用户执行 python main.py
    ↓
加载 .env 配置文件
    ↓
初始化 FastAPI 应用
    ↓
注册路由 (models, service, upload, system)
    ↓
初始化 SQLite 数据库
    ↓
启动 Uvicorn 服务器 (Port 23058)
    ↓
等待 API 请求
```

### 2. **模型下载流程**
```
POST /models/download/start
    ↓
生成 task_id
    ↓
异步启动下载任务
    ↓
通过 huggingface_hub 下载
    ↓
实时捕获下载进度 (tqdm)
    ↓
保存到 models/ 目录
    ↓
记录到数据库 models 表
    ↓
返回下载完成状态
```

### 3. **子服务启动流程**
```
POST /service/start
    ↓
检查子服务状态
    ↓
读取数据库模型列表
    ↓
生成 ModelSettings 配置
    ↓
通过 subprocess 启动 model_service.py
    ↓
model_service.py 创建 llama.cpp 应用
    ↓
加载所有模型到内存/GPU
    ↓
启动推理服务 (Port 23059)
    ↓
提供 OpenAI 兼容 API
```

### 4. **推理请求流程**
```
客户端 → POST /v1/chat/completions (Port 23059)
    ↓
llama.cpp 路由处理
    ↓
选择对应模型 (通过 model 参数)
    ↓
GPU/CPU 推理计算
    ↓
流式或完整响应
    ↓
返回结果给客户端
```

### 5. **分片上传流程**
```
POST /upload/init (初始化)
    ↓
创建上传会话 (task_id)
    ↓
POST /upload/chunk (逐片上传)
    ↓
保存到 data/uploads/
    ↓
记录到 upload_chunks 表
    ↓
POST /upload/complete (完成)
    ↓
合并所有分片
    ↓
移动到 models/ 目录
    ↓
清理临时文件
    ↓
添加到数据库 models 表
```

## 🏛️ 设计模式

### 1. **单例模式 (Singleton)**
- `Config`: 全局配置实例
- `Database`: 数据库连接管理
- `ModelLoader`: 模型加载器
- `ModelManager`: 模型管理器
- `ServiceManager`: 服务管理器
- `UploadManager`: 上传管理器

### 2. **分层架构 (Layered Architecture)**
- **表现层**: FastAPI 路由 (routes/)
- **业务层**: 核心逻辑 (core/)
- **数据层**: 数据库操作 (database.py)
- **存储层**: 文件系统 (models/, data/)

### 3. **依赖注入 (Dependency Injection)**
- FastAPI 的依赖系统
- 配置对象注入

### 4. **异步编程 (Async/Await)**
- 异步路由处理
- 后台任务管理
- 非阻塞 I/O

### 5. **进程隔离 (Process Isolation)**
- 主服务与子服务分离
- 避免模型加载影响管理功能
- 支持独立重启

## 🎯 关键特性

1. **双端口架构**: 管理与推理分离
2. **异步下载**: 非阻塞模型下载，支持进度查询
3. **分片上传**: 支持大文件上传，断点续传
4. **多模型支持**: 同时加载多个模型
5. **GPU加速**: 自动利用GPU资源
6. **OpenAI兼容**: 标准API接口
7. **进程管理**: 子服务独立控制
8. **数据库持久化**: 元数据和状态管理
9. **系统监控**: 实时资源监控
10. **RESTful API**: 标准化接口设计

## 📈 性能优化

- **GPU Offloading**: KQV offloading 降低显存占用
- **并发限制**: 下载任务串行执行，避免资源竞争
- **连接池**: 数据库连接复用
- **异步I/O**: 非阻塞操作提升并发
- **进度缓存**: 限制更新频率，减少I/O开销
- **分片上传**: 降低内存占用，支持断点续传
